Import2 Button

The fastest and easiest way to start providing your customers with importing functionality is via Import2 Button. All you will need to do is to paste an HTML snippet into a page in your app and set data-api-token attribute to your Import2 API token.

Step 2: A Better Experience For Customer

The only configuration value required is data-api-token, which you need to set to your Import2 API token. But we recommend to provide all the required information needed to access API of your tool: username, API token, instance url, etc. This will greatly improve customer experience - we will not ask her to provide this information.

Name Required? Description Example values
data-api-token required Your Import2 API token.
data-destination-instance-url optional Url of the destination tool instance. “http://myblog.tumblr.com/”
“https://mycorp.salesforce.com/”
data-destination-username optional Username used to sign in into the destination tool (or email address if they are used in destination tool) “janeimport”
“jane.import@mycorporation.com”
data-destination-token optional API token used to access destination tool “apitoken”
data-destination-option-xxx_yyy optional If accessing API of destination tool requires additional properties, then you will need to provide them in form of data-destination-option-#{propety_name}. Use ‘_’ (underscore) to separate words in property name.
data-destination-option-organization_id
data-destination-option-refresh_token
